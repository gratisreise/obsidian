


### NoSQL이 뭐지?
관계형의 단점 
- 새로운 컬럼 추가시 스키마를 변경해야한다. => 너무 위험부담이 크다
- 정규화로 인한 복잡한 join => 성능저하
- scale-out에 유연하지 않다
- transacion ACID

### 등장배경
- high-throughput이 요구됨
- law-latency이 요구됨
- 비정형 데이터의 증가(기존의 rdb는 유연성이 떨어짐)


### 특징
- flexible schema
- 중복 허용(join 회피)
- scale-out 최적화:뇌지컬(로직)이 좋을수록 다양하게 가능
